🚀 NewAPI 配置生成器

轻松生成多账号签到配置，同站点多账号只需填写一次 URL

🔑 Secrets 设置 ↗

📋 导入 JSON 配置

粘贴已有的 JSON 配置，一键导入并按站点自动分组填充表单

💡 使用提示：

  • 站点分组：同一站点的 URL 和 CF Clearance 只需填写一次，站点下可添加多个账号
  • 账号信息：每个账号只需填写 Session、用户ID 和备注名称
  • 保存到本地：将配置保存到浏览器本地存储，下次访问自动加载
  • 明文存储：数据以明文形式存储在浏览器中，请注意安全

💡 使用方法：

                    
    
                        
  • GitHub Actions：复制上面的配置，在仓库的 Settings → Secrets → New repository secret 中添加名为 NEWAPI_ACCOUNTS 的 Secret
    • 
                        
  • 本地运行：将配置设置为环境变量 NEWAPI_ACCOUNTS
    • 
                        
  • 格式推荐：JSON 格式支持备注，更易于管理多个站点

💡 提示

                        
    
                            
  • 候选路径按 NewAPI/One API 系默认前端约定排列：/console、/console/personal、/setting、/user/personal
    • 
                            
  • 新标签页会用浏览器自带 Cookie 打开，可直接看到当前账号的密钥/令牌(即 NEWAPI_ACCOUNTS 里的 session)
    • 
                            
  • 站点魔改过前端时，可手动改下拉里的路径

💡 用法

                        
    
                            
  • Base64：复制下方文本 → 新建 GitHub Gist（或任意可匿名 GET 的 raw 文件托管）→ 取 raw URL 填入 GitHub Secret CONFIG_URL，CONFIG_AUTH 留空
    • 
                            
  • 原始 JSON：把 JSON 存到 WebDAV/对象存储，URL 填 CONFIG_URL，CONFIG_AUTH 按 user:password 或 token:xxx 填
    • 
                            
  • data URL：仅本地便捷用直接贴 CONFIG_URL；CI 环境请用 Base64（requests 不支持 data: scheme）
    • 
                            
  • 云端账号配置优先级高于 NEWAPI_ACCOUNTS 环境变量
    • 
                            
  • 本页只生成账号配置；钉钉 / 邮件 / ServerChan 通知请在 GitHub Secrets 里单独配（见上方 .env 导出页的占位变量名）
    • 
                        

                    

                


                
                

                    

                        生成可直接本地使用的 .env 文本。账号走单行 JSON（被 load_env_file 按行读取，多行会被切断，故强制单行）。
                    

                    

                        
                    

                    

                        
                    

                    

                        
                    

                    

                        
                        
                        

                    

                    

                        
💡 重要

                        
    
                            
  • NEWAPI_ACCOUNTS= 后不加外层引号，load_env_file 不剥引号，加了会让首尾 " 进入 session 导致认证失败
    • 
                            
  • 含敏感信息，勿提交 git（仓库根 .gitignore 已含 .env）
    • 
                            
  • 通知配置默认注释，需用时取消注释并填值
